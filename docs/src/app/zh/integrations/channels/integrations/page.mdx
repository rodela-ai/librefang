# 集成与协议渠道

## LINE

### 前置条件

- LINE Messaging API 频道（从 [LINE Developers Console](https://developers.line.biz/en/docs/messaging-api/) 创建）
- 可公网访问的 webhook URL

### 设置步骤

1. 前往 [LINE Developers Console](https://developers.line.biz/console/)，创建新的 Messaging API 频道。
2. 记录 **Channel Secret** 并签发 **Channel Access Token**。
3. 设置环境变量：

```bash
export LINE_CHANNEL_SECRET=your-secret        # 环境变量
export LINE_ACCESS_TOKEN=your-access-token
librefang vault set LINE_CHANNEL_SECRET       # 加密金库（推荐）
librefang vault set LINE_ACCESS_TOKEN
librefang config set-key line                 # .env 文件
# 或通过仪表板 "Set API Key" 按钮设置          # secrets.env
```

4. 添加到配置文件：

```toml
[channels.line]
channel_secret_env = "LINE_CHANNEL_SECRET"
access_token_env = "LINE_ACCESS_TOKEN"
default_agent = "assistant"
```

5. 在 LINE Developers Console 中将 webhook URL 设置为 LibreFang 实例的地址（例如 `https://your-domain.com:4545/channels/line/webhook`）。
6. 重启守护进程。

### 工作原理

LINE 适配器在主 API 服务器（默认端口 4545）上注册 webhook 路由，接收来自 LINE Messaging API 的消息事件。每个请求通过 Channel Secret 的 HMAC-SHA256 签名进行验证。响应通过 LINE Reply API 或 Push API 发送。

## Viber

### 前置条件

- Viber Bot 令牌（从 [Viber 开发者门户](https://developers.viber.com/docs/api/) 获取）
- 可公网访问的 webhook URL

### 设置步骤

1. 在 [Viber Admin Panel](https://partners.viber.com/) 创建 Bot 账户。
2. 复制认证令牌。
3. 设置环境变量：

```bash
export VIBER_AUTH_TOKEN=your-viber-auth-token  # 环境变量
librefang vault set VIBER_AUTH_TOKEN           # 加密金库（推荐）
librefang config set-key viber                # .env 文件
# 或通过仪表板 "Set API Key" 按钮设置          # secrets.env
```

4. 添加到配置文件：

```toml
[channels.viber]
auth_token_env = "VIBER_AUTH_TOKEN"
webhook_url = "https://your-domain.com:4545/channels/viber/webhook"
default_agent = "assistant"
```

5. 重启守护进程。适配器在启动时自动向 Viber 注册 webhook URL。

### 工作原理

Viber 适配器在启动时向 Viber API 注册 webhook，并通过 HTTPS POST 回调接收消息事件。响应通过 Viber REST API 的 `send_message` 端点发送。

## Facebook Messenger

### 前置条件

- Meta（Facebook）开发者账户，拥有 Facebook 主页和应用（参见 [Messenger Platform 文档](https://developers.facebook.com/docs/messenger-platform/)）
- 可公网访问的 webhook URL

### 设置步骤

1. 前往 [Meta for Developers](https://developers.facebook.com/)，创建或选择一个应用。
2. 为应用添加 **Messenger** 产品。
3. 为 Facebook 主页生成 **Page Access Token**。
4. 选择一个 **Verify Token**（你自定义的任意字符串）。
5. 设置环境变量：

```bash
export MESSENGER_PAGE_TOKEN=your-page-token   # 环境变量
export MESSENGER_VERIFY_TOKEN=your-verify-token
librefang vault set MESSENGER_PAGE_TOKEN      # 加密金库（推荐）
librefang vault set MESSENGER_VERIFY_TOKEN
librefang config set-key messenger            # .env 文件
# 或通过仪表板 "Set API Key" 按钮设置          # secrets.env
```

6. 添加到配置文件：

```toml
[channels.messenger]
page_token_env = "MESSENGER_PAGE_TOKEN"
verify_token_env = "MESSENGER_VERIFY_TOKEN"
default_agent = "assistant"
```

7. 在 Meta 开发者面板中设置 webhook 回调 URL（例如 `https://your-domain.com:4545/channels/messenger/webhook`），输入 verify token，并订阅 `messages` 事件。
8. 重启守护进程。

### 工作原理

Messenger 适配器在主 API 服务器（默认端口 4545）上注册 webhook 路由，处理来自 Messenger Platform 的 webhook 验证（GET）和传入消息事件（POST）。响应通过 Send API 使用 page access token 发送。

## 钉钉

支持两种接收模式：
- **Stream 模式**（默认，推荐）：通过钉钉 Stream 协议建立 WebSocket 长连接。无需公网 IP 或端口。
- **Webhook 模式**（传统）：HTTP 回调服务器。需要公网可访问的 URL。

### 前置条件

- 在[钉钉开放平台](https://open.dingtalk.com/document/)创建的应用
- **Stream 模式**：需要应用的 App Key 和 App Secret
- **Webhook 模式**：需要自定义机器人的 webhook 访问令牌和签名密钥

### 设置步骤（Stream 模式 -- 推荐）

1. 前往[钉钉开放平台](https://open.dingtalk.com/)，创建应用（或使用现有应用）。
2. 启用 **机器人** 能力，记录 **App Key**（Client ID）和 **App Secret**（Client Secret）。
3. 设置环境变量：

```bash
export DINGTALK_APP_KEY=your-app-key          # 环境变量
export DINGTALK_APP_SECRET=your-app-secret
librefang vault set DINGTALK_APP_KEY          # 加密金库（推荐）
librefang vault set DINGTALK_APP_SECRET
```

4. 添加到配置文件：

```toml
[channels.dingtalk]
receive_mode = "stream"
app_key_env = "DINGTALK_APP_KEY"
app_secret_env = "DINGTALK_APP_SECRET"
default_agent = "assistant"
```

5. 重启守护进程。

### 设置步骤（Webhook 模式）

1. 在钉钉中，打开群聊，前往 **群设置 > 智能群助手 > 添加机器人**。
2. 选择 **自定义机器人**，启用 **安全设置**（建议使用签名模式）。
3. 记录 webhook 访问令牌和签名密钥。
4. 设置环境变量：

```bash
export DINGTALK_ACCESS_TOKEN=your-token       # 环境变量
export DINGTALK_SECRET=your-signing-secret
librefang vault set DINGTALK_ACCESS_TOKEN     # 加密金库（推荐）
librefang vault set DINGTALK_SECRET
```

5. 添加到配置文件：

```toml
[channels.dingtalk]
receive_mode = "webhook"
access_token_env = "DINGTALK_ACCESS_TOKEN"
secret_env = "DINGTALK_SECRET"
default_agent = "assistant"
```

6. 重启守护进程。

### 工作原理

在 **Stream 模式** 下，适配器向钉钉网关 API（`/v1.0/gateway/connections/open`）注册以获取 WebSocket 端点和票据，然后维持持久的 WebSocket 连接，接收机器人消息回调并响应 ACK 帧。心跳 ping/pong 自动处理。

在 **Webhook 模式** 下，适配器通过 HTTP 回调接收来自钉钉机器人 API 的消息。每个请求使用配置的密钥通过 HMAC-SHA256 签名进行验证。响应通过带有签名时间戳的钉钉 webhook URL 发送。

## 企业微信

### 前置条件

- 具有**智能机器人**的企业微信账户（从[企业微信管理后台](https://work.weixin.qq.com/)创建）

### 设置步骤

1. 在企业微信管理后台，前往 **应用管理 > 创建应用**，选择**智能机器人**。
2. 记录 **Bot ID** 和 **Secret**。
3. 设置密钥环境变量：

```bash
export WECOM_BOT_SECRET=your-bot-secret       # 环境变量
librefang vault set WECOM_BOT_SECRET          # 加密金库（推荐）
librefang config set-key wecom                # .env 文件
# 或通过仪表板 "Set API Key" 按钮设置          # secrets.env
```

4. 添加到配置文件：

```toml
[channels.wecom]
bot_id = "YOUR_BOT_ID"
secret_env = "WECOM_BOT_SECRET"
# account_id = "my-bot"          # 可选：多 Bot 路由的唯一标识
# default_agent = "assistant"    # 可选：将消息路由到指定 Agent
```

5. 重启守护进程。

### 工作原理

企业微信适配器通过持久 WebSocket 连接到 `wss://openws.work.weixin.qq.com`。使用 Bot ID 和签名令牌认证后，通过 `aibot_subscribe` 订阅消息。用户消息以 `aibot_msg_callback` JSON 帧到达，回复通过 `aibot_send_msg` 帧发送。无需公网 HTTP 端点或 AES 加解密。

## IRC

### 前置条件

- IRC 服务器地址和 Bot 昵称（协议定义见 [RFC 2812](https://datatracker.ietf.org/doc/html/rfc2812)）

### 设置步骤

1. 选择 IRC 服务器和要加入的频道。
2. 设置环境变量：

```bash
export IRC_SERVER=irc.libera.chat:6697        # 环境变量
export IRC_NICK=librefang-bot
export IRC_CHANNELS=#channel1,#channel2
# export IRC_PASSWORD=your-password           # 可选
librefang vault set IRC_SERVER                # 加密金库（推荐）
librefang vault set IRC_NICK
librefang vault set IRC_CHANNELS
# librefang vault set IRC_PASSWORD            # 可选
librefang config set-key irc                  # .env 文件
# 或通过仪表板 "Set API Key" 按钮设置          # secrets.env
```

3. 添加到配置文件：

```toml
[channels.irc]
server_env = "IRC_SERVER"
nick_env = "IRC_NICK"
channels = ["#channel1", "#channel2"]
# password_env = "IRC_PASSWORD"   # optional
default_agent = "assistant"
```

4. 重启守护进程。

### 工作原理

IRC 适配器通过 TCP（可选 TLS）连接到 IRC 服务器，进行认证并加入配置的频道。它监听 PRIVMSG 事件并路由到配置的 Agent。响应以 PRIVMSG 回复形式发送。

## MQTT

### 前置条件

- LibreFang 主机可访问的 MQTT broker（例如 Mosquitto、HiveMQ、EMQX），参见 [MQTT 规范](https://mqtt.org)
- 编译时需要启用 `all-channels` feature flag

### 设置步骤

1. 确保 MQTT broker 已运行且可访问。
2. 设置环境变量（可选，用于需要认证的 broker）：

```bash
export MQTT_BROKER_URL=mqtt://broker:1883     # 环境变量
export MQTT_TOPIC=home/agents/input
# export MQTT_USERNAME=your-username          # 可选
# export MQTT_PASSWORD=your-password          # 可选
librefang vault set MQTT_BROKER_URL           # 加密金库（推荐）
librefang vault set MQTT_TOPIC
# librefang vault set MQTT_USERNAME           # 可选
# librefang vault set MQTT_PASSWORD           # 可选
librefang config set-key mqtt                 # .env 文件
# 或通过仪表板 "Set API Key" 按钮设置          # secrets.env
```

3. 添加到配置文件：

```toml
[channels.mqtt]
host = "broker.example.com"
port = 1883
client_id = "librefang-bot"
subscribe_topics = ["home/agents/input", "home/agents/commands"]
response_topic = "home/agents/output"
qos = "at_least_once"                # at_most_once | at_least_once | exactly_once
# username_env = "MQTT_USERNAME"     # optional
# password_env = "MQTT_PASSWORD"     # optional
keep_alive_secs = 30
default_agent = "home-assistant"
```

4. 重启守护进程。

### 工作原理

MQTT 适配器使用 `rumqttc` crate 通过 MQTT v3.1.1/v5 连接到 broker。它订阅配置的主题接收传入消息，并将 Agent 响应发布到配置的响应主题。内置自动重连和指数退避机制。

## Webhook

### 前置条件

- 可接受 HTTP POST 请求的目标 URL（无需注册外部服务）

### 设置步骤

1. 选择 webhook 负载要发送到的 URL。
2. 选择用于 HMAC-SHA256 签名验证的共享密钥。
3. 设置环境变量：

```bash
export WEBHOOK_URL=https://your-service.example.com/hook  # 环境变量
export WEBHOOK_SECRET=your-shared-secret
librefang vault set WEBHOOK_URL               # 加密金库（推荐）
librefang vault set WEBHOOK_SECRET
librefang config set-key webhook              # .env 文件
# 或通过仪表板 "Set API Key" 按钮设置          # secrets.env
```

4. 添加到配置文件：

```toml
[channels.webhook]
url_env = "WEBHOOK_URL"
secret_env = "WEBHOOK_SECRET"
default_agent = "assistant"
```

5. 重启守护进程。

### 工作原理

Webhook 适配器提供通用的双向 HTTP 集成。传入消息以 HTTP POST 请求形式发送到 LibreFang 的 webhook 端点，通过共享密钥的 HMAC-SHA256 进行验证。响应以 HTTP POST 负载形式送达配置的 URL，同样使用 HMAC-SHA256 签名。

### `deliver_only` 透传模式

Webhook 通道还可以跑成零 LLM 透传:入站 payload 完全绕过 agent 循环(无 sanitiser、无限流、无 agent 查找),直接转发到目标通道。适合推送通知和带外告警 —— 你只想让消息落到 Telegram / Slack 之类的地方。

```toml
[channels.webhook]
enabled = true
deliver_only = true
deliver = "telegram:123456789"     # deliver_only = true 时必填
```

`deliver_only = true` 但 `deliver` 未设时,librefang 启动时打 `warn`,webhook 保持不激活。完整的 `__deliver_only__` / `__deliver_target__` 内部 metadata 信号化机制见 channels overview 的 [Webhook `deliver_only` 模式](/zh/integrations/channels#webhook-deliver_only-mode)。

## Voice（语音）

### 前置条件

- 兼容 OpenAI 的 STT/TTS 服务 API key（如 OpenAI、Groq 或自建 Whisper 实例）

### 设置步骤

1. 设置 STT/TTS API key：

```bash
export OPENAI_API_KEY=sk-...
librefang vault set OPENAI_API_KEY              # 加密保险库（推荐）
```

2. 添加配置：

```toml
[channels.voice]
api_key_env = "OPENAI_API_KEY"
stt_url = "https://api.openai.com"
tts_url = "https://api.openai.com"
tts_voice = "alloy"
# buffer_threshold = 32768    # 触发 STT 的音频缓冲区大小（字节）
# default_agent = "assistant"
```

3. 重启守护进程。

### WebSocket 协议

客户端连接 `ws://your-host:4545/channels/voice/ws`，通过二进制帧和文本帧交互：

**客户端 -> 服务端（二进制）：** 原始音频数据（PCM 16-bit LE 单声道 16kHz 或 Opus 包）。

**客户端 -> 服务端（JSON）：**
```json
{ "type": "config", "format": "pcm", "sample_rate": 16000 }
{ "type": "end_of_speech" }
```

**服务端 -> 客户端（JSON）：**
```json
{ "type": "ready" }
{ "type": "transcript", "text": "..." }
{ "type": "response", "text": "..." }
{ "type": "error", "message": "..." }
```

**服务端 -> 客户端（二进制）：** 合成的语音音频数据。

### 工作原理

Voice 适配器通过共享 API 服务器上的 `/channels/voice/ws` 端点接受实时 WebSocket 音频流。音频数据在缓冲区达到阈值（默认 32KB）后，通过 STT 服务（默认 OpenAI Whisper API）进行语音转文字。转写后的文本通过 channel bridge 路由到 Agent，Agent 的文字回复可通过 TTS 合成为语音并流式返回客户端。支持 PCM（16-bit LE 单声道）和 Opus 音频格式。

| 配置字段 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `api_key_env` | string | `"OPENAI_API_KEY"` | STT/TTS 服务 API key 的环境变量名。 |
| `stt_url` | string | `"https://api.openai.com"` | STT（语音转文字）API 基础 URL。 |
| `tts_url` | string | `"https://api.openai.com"` | TTS（文字转语音）API 基础 URL。 |
| `tts_voice` | string | `"alloy"` | TTS 语音名称。可选：`alloy`、`echo`、`fable`、`onyx`、`nova`、`shimmer`。 |
| `buffer_threshold` | usize | `32768` | 触发 STT 的音频缓冲区大小（字节）。 |
| `default_agent` | string | - | 语音消息路由到的默认 Agent 名称。 |
